【操作系统】MIT 6.828/6.S081 2019 Lab1

Posted on 2020-12-28 In Labs

听闻嘟嘟子的操作系统实验课内容是做MIT的操作系统课的lab，自己也来了兴趣，能做多少做多少吧！
一开始我没有仔细看xv6（实验用到的系统）的手册，也没怎么看指导书，拿来就做，实在是有些蠢蛋，不可避免地参考了网上其他人的资料。虽然我一直借鉴到了lab2，但心里一想感觉不是滋味，又不是在应付作业，抄别人的还不如自己写的收获大。在下的确是愚笨至极，lab1搞了两三天才弄出来，不过收获是真的挺多，也顺带复习了自己用C语言编写代码的能力，针不戳！

下面就是我在编写lab1相关代码的时候的一些备忘录，当然了，全部代码都会有。lab1主要是熟悉xv6这个利用精简指令集的类unix系统，编写sleep、pingpong、primes、find、xargs这5个指令的源代码。在写的过程中，我发现难度是逐渐上升的，不由得感慨MIT就是MIT。

sleep.c

  • 使用 atoi 把字符串转化成数字
  • 使用 sleep 这个系统调用来实现sleep的功能
  • 注意在 main 函数中使用 exit() 退出程序
  • 注意输出 error 信息

pingpong.c

  • 使用 pipe 来建立管道
  • 使用 fork 来创建子进程
  • 使用 read 从管道中读取数据，write 从管道中写入数据
  • 个人认为这个程序的目的是让我们熟悉以上这四个系统调用函数，熟悉pipe的使用方法

primes.c

  • 个人认为这个程序是对上面pingpong程序中所使用的四个系统调用的综合应用，当然也用到了一些诸如管道重定向的方法，具体在xv6的手册中对wc程序的核心代码解释中出现
  • 利用管道来并发输出素数的方法主要参照的是Doug Mcllroy的素数筛方法，左侧的进程把数字传递给右侧的进程，右侧的进程通过验证该数字是否能被已有的素数整除来去掉那些非素数，再把素数传递给后续的进程打印
  • 注意关闭进程不需要的文件描述符，即需要重定向
  • 注意将进程关闭干净，当write管道关闭后read管道将会返回EOF，可以利用这一点结束循环

find.c

  • 参照 ls.c 来读取文件夹
  • 利用递归在文件夹中的子文件夹查找文件
  • 不要递归进入 “.” 和 “..”，不然停不下来
  • 支持用正则表达式匹配字符串，参照 grep.c 当中的正则匹配器
  • 个人认为这个程序最麻烦的地方是理解 find 的过程以及正则匹配器的工作过程

xargs.c

  • 需要理解什么是 xargs。我们知道管道命令可以将一个命令的标准输入转化为另一个命令的标准输入。但有的命令并不接受标准输入作为参数，只接受在命令行输入的参数作为参数，这导致无法用管道命令来传递参数。而xargs命令的作用就是将标准输入转化为命令行参数。在管道命令与其他命令一起时候在时候，在管道命令与真正要执行的命令之间可以使用xargs命令，将管道命令所得到的标准输入转化为命令行参数
  • xargs命令也可以单独使用。输入xargs按下回车以后，命令行就会等待用户输入，作为标准输入，然后ctrl+d，表示输入结束。但xv6与unix系统下的xargx的不同：根据lab中的使用例子可以看出，xv6的xargs每次回车都会执行一次命令并输出结果，直到ctrl+d时结束；而linux中的实现则是一直接收输入，收到ctrl+d时才执行命令并输出结果
  • 需要使用 fork 和 exec 的系统调用来对每一行输入执行指令，在父进程记得使用 wait 等待子进程的结束
  • 每次只读1个字符，遇到换行符时停下执行命令
  • 在 kernel/param.h 里面有声明 MAXARG 按需使用
